1724 @ingroup THREADPRIVATE
1725 @param loc source location information
1726 @param gtid global thread number
1727 @param cpy_size size of the cpy_data buffer
1728 @param cpy_data pointer to data to be copied
1729 @param cpy_func helper function to call for copying data
1730 @param didit flag variable: 1=single thread; 0=not single thread
1731 
1732 __kmpc_copyprivate implements the interface for the private data broadcast needed for
1733 the copyprivate clause associated with a single region in an OpenMP<sup>*</sup> program (both C and Fortran).
1734 All threads participating in the parallel region call this routine.
1735 One of the threads (called the single thread) should have the <tt>didit</tt> variable set to 1
1736 and all other threads should have that variable set to 0.
1737 All threads pass a pointer to a data buffer (cpy_data) that they have built.
1738 
1739 The OpenMP specification forbids the use of nowait on the single region when a copyprivate
1740 clause is present. However, @ref __kmpc_copyprivate implements a barrier internally to avoid
1741 race conditions, so the code generation for the single region should avoid generating a barrier
1742 after the call to @ref __kmpc_copyprivate.
1743 
1744 The <tt>gtid</tt> parameter is the global thread id for the current thread.
1745 The <tt>loc</tt> parameter is a pointer to source location information.
1746 
1747 Internal implementation: The single thread will first copy its descriptor address (cpy_data)
1748 to a team-private location, then the other threads will each call the function pointed to by
1749 the parameter cpy_func, which carries out the copy by copying the data using the cpy_data buffer.
1750 
1751 The cpy_func routine used for the copy and the contents of the data area defined by cpy_data
1752 and cpy_size may be built in any fashion that will allow the copy to be done. For instance,
1753 the cpy_data buffer can hold the actual data to be copied or it may hold a list of pointers
1754 to the data. The cpy_func routine must interpret the cpy_data buffer appropriately.
1755 
1756 The interface to cpy_func is as follows:
1757 @code
1758 void cpy_func( void *destination, void *source )
1759 @endcode
1760 where void *destination is the cpy_data pointer for the thread being copied to
1761 and void *source is the cpy_data pointer for the thread being copied from.
1762 */

1771

1773 
1774  if ( __kmp_env_consistency_check ) {
1775  if ( loc == 0 ) {
1776  KMP_WARNING( ConstructIdentInvalid );
1777  }
1778  }
1779
